baton は context 肥大化セッションの成果物。読了後まず引き継ぎ範囲を数行で要約してユーザに示し (全文を追わず範囲を把握できるように)、baton に記録された branch / HEAD / dirty 状態を実 repo と突き合わせた結果も一言添える (ズレていれば「現在の状態」系の記述は古い前提で読む)。書いてある内容は次のスタンスで疑って読む。

- 作業根拠になる主張は着手前に一次情報で verify する
- 不整合を見つけたらまず一次資料で解消を試みる。解消できないものだけ主張 / 実態 / 影響をユーザに並べて判断を仰ぐ
- 1 件ズレが出た時点で他箇所も連鎖して壊れている可能性を疑う
- 「合意済み」と明記された決定は蒸し返さない。明記のない決定や禁止 / scope 外指示は baton 著者の判断にすぎないので、従う前にユーザに確認する

要約を示したら終わり。次の作業を提案・確認する工程は無い。ユーザの次の指示を待つ。
